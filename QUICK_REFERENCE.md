# 🎯 YARA-X Manager 快速参考

## 🚀 快速启动

### 一键启动（推荐）
```powershell
.\start-all.ps1
```
✅ 自动启动前后端  
✅ 独立窗口运行  
✅ 自动打开浏览器

### 访问地址
- **前端界面**: http://localhost:3000
- **后端 API**: http://localhost:8000
- **API 文档**: http://localhost:8000/docs

---

## 📱 界面功能

### 1. 📊 仪表板
- 4 个统计卡片（总任务/文件/威胁/安全）
- 检测率分析图表
- 系统活动时间线
- 最近检测记录

### 2. 📂 文件扫描
- **拖拽上传**：支持批量文件
- **实时进度**：0-100% 进度条
- **状态显示**：
  - 🔄 上传中（蓝色）
  - ✅ 完成（绿色）
  - ❌ 失败（红色）
- **检测结果**：
  - 🟢 安全
  - 🔴 检测到威胁

### 3. 📜 规则管理
- 创建/编辑/删除 YARA 规则
- 上传 .yar 文件
- 规则状态管理
- 匹配统计

## 🔧 常用操作

### 上传扫描文件
1. 打开"文件扫描"页面
2. 拖拽文件到上传区
3. 查看实时进度
4. 查看扫描结果

### 查看统计
1. 打开"仪表板"页面
2. 查看统计卡片
3. 分析检测率图表
4. 浏览最近记录

### 管理规则
1. 打开"规则管理"页面
2. 点击"创建规则"或"上传文件"
3. 填写/编辑规则信息
4. 保存并启用

## ⚡ 快捷提示

### 进度显示
- **0-90%**：文件上传中
- **90-100%**：YARA 扫描中
- **100%**：扫描完成

### 状态标签
- **⚪ pending**：等待中
- **🔵 running**：执行中
- **🟢 completed**：已完成
- **🔴 failed**：失败

### 自动功能
- ✅ 自动刷新数据
- ✅ 自动移除完成项（3秒）
- ✅ 自动显示威胁警告

## 🐛 常见问题

### Q: 前端无法连接后端？
**A**: 确保后端已启动：
```powershell
cd backend
python main.py
```

### Q: 上传文件无响应？
**A**: 检查：
- 后端服务状态
- 文件格式是否支持
- 文件大小限制

### Q: 数据不更新？
**A**: 
- 刷新浏览器（F5）
- 检查自动刷新设置
- 重启前端服务
